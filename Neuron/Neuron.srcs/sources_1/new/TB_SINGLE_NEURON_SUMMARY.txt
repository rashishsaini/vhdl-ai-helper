================================================================================
ROOT CAUSE ANALYSIS
================================================================================

The failing tests share a common issue: WEIGHT PERSISTENCE ACROSS TESTS

The testbench assumes weights are independent between tests, but:
- Weight update in Test 6 modifies weights
- Test 7 uses modified weights instead of fresh 1.0 values  
- Test 8 initialization happens while UPDATE FSM is busy
- Test 10 uses weights from previous tests

VHDL DESIGN INSIGHT:
The single_neuron.vhd has weight_init_en checked inside UPD_IDLE state.
If weight initialization is attempted while upd_state != UPD_IDLE, the
write is IGNORED.
