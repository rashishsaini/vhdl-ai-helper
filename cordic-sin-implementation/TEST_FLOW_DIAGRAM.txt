                        CONCURRENT PROCESSES
================================================================================

While main test executes, these run in parallel:

┌─────────────────────────────────────────────────────────────┐
│  CLOCK GENERATION PROCESS                                   │
│  • Runs continuously until test_complete                    │
│  • Period: 10 ns (100 MHz)                                  │
│  • Provides timing reference                                │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  FSM MONITOR PROCESS                                        │
│  • Tracks ready/done signals                                │
│  • Infers current FSM state                                 │
│  • Reports state transitions                                │
│  • Counts cycles per state                                  │
│  • Output: "[FSM] IDLE -> INIT detected"                    │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  CONCURRENT ASSERTION 1                                     │
│  • Always checks: done = valid                              │
│  • Reports immediately if mismatch                          │
│  • Severity: error                                          │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  CONCURRENT ASSERTION 2                                     │
│  • Checks: ready='0' during computation                     │
│  • Tracks computing_detected flag                           │
│  • Reports protocol violations                              │
│  • Severity: error                                          │
└─────────────────────────────────────────────────────────────┘
